1.Что такое HTTP и HTTPS.
HTTP-(HyperText transfer protocol-протокол передачи гипертекста-это прикладной протокол передачи данных
в сети.Основан на использовании технологии "клиент-сервер".Используется для получения информации с веб-сайтов.
HTTPS-(HyperText Transfer Protocol Secure)-безопасный протокол передачи гипертекста,это расширение протокола HTTP.
Поддерживающее шифрование посредством криптографических протоколов SSL и TLS.

2.HTTP методы.
GET-получение информации с сервера (в URL).Только чтение,кешируется.
POST-добавление информации на сервер(в Body).Создание,обновление,удаление.
PUT-замена,изменение информации на сервере.Создание,обновление.
DELETE-удаление информации с сервера.Удаление.
OPTIONS-получение сведений с сервера.Не кешируется.
HEAD-получение информации с сервера,проверка URL.

3.HTTP статус коды сервера -
100 - информационные сообщения.
200 - запрос прошел успешно (OK).
300 - перенаправление 
301 - страница находится по новому адресу.
302 - запрошенный ресурс перемещен,но был найден.
400 - ошибка на стороне клиента.
401- не авторизован.
403 - запрещенный запрос.
404 - страница не найдена.
405 - метод не разрешен.
406 - неприемлемый ответ.
408 - время ожидания сервером поступления остальной части запроса из браузера истекло.
410 - запрещенный ресурс отсутствует и не будет возвращен.
429 - слишком много запросов.
500 - ошибки на стороне сервера.
501 - не реализовано .Ошибка со стороны хостинг -провайдера.
502 - некорректный шлюз.
503 - вданный момент сервер не может обработать запрос.
504 - сервер,действующий как шлюз,не смог дождаться ответа от другого сервера.

5.Что такое ядро браузера.
Состоит из оболочки и ядра.Ядро делится на движок рендеринга,и движок JS.
Движок JS - анализирует язык javascript и выполняет язык javascript для достижения
динамических эффектов веб-страницы.(Динамика).
Движок рендеринга - отвечает за содержимое веб-страницы(HTML,XML,СSS изображения и т.д.)
Как браузер отображает содержимое веб-страницы и информацию о формате страницы.

7.Что такое API- Application programming interface - программный интерфейс приложения - 
при посещении пользователем какой - либо страницы в сети, он взаимодействует с API 
удаленного сервера. API - это составляющая часть сервера,которая получает запросы и отправляет
ответы.
Например: (Механника) Facebook.com на удаленный сервер Facebook отправляется соответствующий запрос
как только браузер получает ответ, то интерпретирует код и отображает страницу (используя ядро
JS + рендеринг)

8.Что такое endpoint - это само обращение к маршруту отдельным HTTP методом.Выполняет конкретную задачу,
принимает параметра и возвращает данные клиенту.

9.URI- (uniform resourse identificator)-уникальный идентефикатор.
  URN- (uniform resourse name)-уникальное имя ресурса.
  URL- (uniform resourse locator)- уникальный адрес местанахождения в сети.

10.Идемпотентный HTTP метод - это когда повторный запрос, сделанный один или несколько раз подряд, 
имеет один и тот же эффект, не изменяющий состояние сервера.
GET,HEAD,PUT и DELETE идемпотентны,как и все безопасные методы.

11.Метод HTTP является безопасным, если он не меняет состояние сервера. Безопасный метод проводит операции
только чтение (read - only).
Например:GET,HEAD,OPTIONS.
PUT и DELETE - небезопасные.

12.Идентификация - определение пользователя в системе по имени.
   Аутентификация - определение пользователя в системе по password.
   Авторизация - процесс представления или проверки полномочий на выполнение тех или иных действий в системе.

13.Что такое IP - это уникальный адрес, идентифицирующий устройство в интернете или локальной сети.(интернет-протокол).
Бывает статический( не меняется в конце каждой сессии) и динамический (меняется в конце каждой сессии).

14.Что такое DNS и октаты.
DNS- это технология, которая позволяет браузеру найти запрошенный пользователем сайт по его имени.
DNS сервер - это специализированный компьютер (или группа), который хранит IP- адреса сайтов.
Принцип схож с телефонной книгой сохранения контактов в телефоне.

15.Что такое порт - это числовой идентификатор программы или процесса, которые обслуживают сетевые соединения
на заданном сетевом адресе(IP-адресе).
На Linux 65536 портов.
 
16.КЕШ - CASH специальная область на диске, предназначенная для временного хранения информации и для часто 
используемых данных и команд.

17.Куки - cookie-это текстовые файлы, которые компьютер загружает в память с веб-страниц.(имя пользователя,password).

18.Уровни OSI 
   1.Прикладной уровень (application layer)
   2.Уровень представления (presentation layer)
   3.Сеансовый уровень (session layer)
   4.Транспортный уровень (transport layer)
   5.Сетевой уровень (network layer)
   6.Канальный уровень( data link layer)
   7.Физический уровень (Physical layer)
19.HTTP-headers - это строки в HTTP сообщении,содержащие разделенную двоеточием пару имя-значение.
